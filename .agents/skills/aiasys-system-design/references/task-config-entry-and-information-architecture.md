# 任务配置入口与信息架构

状态: 主线专题
更新时间: 2026-04-22
适用版本: v0.3.9 当前主线

---

## 1. 文档目标

本文档用于回答三个实现前必须先收口的问题：

1. 当前任务的任务配置应该从哪里进入
2. 任务配置内部应该按什么信息层次组织
3. 不同生命周期阶段下，哪些内容允许编辑

本文档属于系统设计与前端信息架构文档，不替代后端配置模型。

---

## 2. 核心结论

### 2.1 当前主线没有单一“任务配置”总入口

当前代码已经把原来容易混在一起的“任务配置”拆成了两个稳定入口：

- `当前分支设置`
- `工作区设置`

也就是说，当前主线并不是一个统一的大型“任务配置”弹窗，而是按作用域拆开的两个弹窗和若干快捷入口。

### 2.2 当前稳定入口分布

当前代码里的主入口分布是：

- 顶部栏
  - 稳定放 `当前分支设置`
- 输入框右侧工具区
  - 提供 `工作区设置` 快捷入口
- 左侧侧栏工具区 / 折叠菜单
  - 提供 `工作区设置`
  - 提供偏好、模型配置、资源管理、定时器管理、托管概览等旁路入口
- 右侧工作区上下文面板
  - 提供 `工作区设置`

因此如果文档要描述当前真相，不能再把“顶部栏里的任务配置按钮”写成已经存在的统一入口。

### 2.3 当前入口仍然不承接市场目录

任务配置弹窗只负责当前任务覆盖。

市场目录仍然独立存在，例如：

- Skill 市场
- MCP 市场
- 运行环境目录

任务配置里可以展示已启用摘要和跳转动作，但不应把市场整页塞进弹窗。

---

## 3. 当前入口布局

### 3.1 顶部栏

当前顶部栏实际上只稳定承接：

1. 当前工作区 / 当前分支标题摘要
2. `当前分支设置`
3. 托管运行中状态提示

顶部栏当前并没有一个统一的“任务配置”按钮。

### 3.2 左侧栏职责

左侧栏继续承接：

- 新建任务
- 工作区切换
- 工作区工具入口
- 全局入口

左侧栏当前已经直接承接：

- 工作区设置
- 资源管理
- 偏好
- 模型配置
- 定时器管理
- 托管概览

因此它在当前代码里已经不只是“全局导航”，而是工作区级工具入口聚合区。

---

## 4. 当前弹窗结构

### 4.1 当前分支设置弹窗

当前代码里的 `当前分支设置` 主要承接：

- 当前分支运行态摘要
- 历史压缩
- 分支记录查看入口
- 当前分支模型覆盖
- 当前分支专家启用策略
- 当前分支私有记忆

它当前不承接：

- 工作区标题与描述
- 工作区资源挂载
- Skill / 能力目录管理
- 当前分支运行态重置

### 4.2 工作区设置弹窗

当前代码里的 `工作区设置` 已经是一个左侧分组导航 + 右侧内容区的弹窗，主要分组包括：

- basics
- skills
- experts
- capabilities
- resources
- automation

它当前承接的内容包括：

- 工作区标题与描述
- 兼容预置摘要
- 工作区默认执行方式摘要
- Skill 挂载
- 工作区专家目录
- 能力目录与运行时可用性摘要
- 数据库 / 知识库 / 知识图谱挂载
- 自动化与定时任务
- 工作区共享记忆

### 4.3 资源管理入口

数据库、知识库、知识图谱除了在工作区设置中出现摘要和挂载面板，还保留各自的资源管理弹窗。

因此当前主线是：

- 工作区设置负责“当前工作区挂了什么”
- 资源管理弹窗负责“资源本体怎么管理”

---

## 5. 当前入口下的可编辑性

### 5.1 当前分支设置

当前分支设置只有在已经进入某个分支后才可用。

它当前允许用户处理的主要是：

- 模型覆盖
- 专家启用策略
- 分支私有记忆
- 历史压缩
- 分支记录查看

### 5.2 工作区设置

工作区设置要求当前已经处于某个工作区上下文中。

当前如果没有工作区上下文，弹窗会明确提示“请先进入一个工作区分支”之类的不可用状态，而不是假装存在一份全局任务配置。

### 5.3 运行中状态

当前代码里，运行中时仍允许打开这些入口，但很多配置修改只会在下一次执行生效。

因此当前文档更适合写“运行中时如何提示保存与生效”，而不是写一套并未落地的 `draft / active / completed` 编辑矩阵。

---

## 6. 前端展示要求

### 6.1 摘要优先

当前这组弹窗默认都应先展示摘要，不默认把所有说明一次性摊开。

每个分组标题建议自带摘要，例如：

- Agent：profile、提示词版本、工具数
- 能力：Skill 数、MCP 数、异常数
- 环境：当前环境、runtime 状态
- 资源：文件数、数据库数、知识库数

### 6.2 文案只说用户需要知道的事

允许的文案：

- 已保存，下次执行生效
- 当前运行中，稍后可切换
- 当前环境需要重建

不应把内部实现解释直接渲染成用户提示。

### 6.3 页面与弹窗职责分离

全局设置页面负责：

- 用户默认值
- 市场目录
- 系统级说明

当前任务相关弹窗负责：

- 当前任务这次怎么用

---

## 7. 当前结论

1. 当前代码已经形成“双入口”结构，不应继续把文档写成单一“任务配置”总入口。
2. 顶部栏当前只承接 `当前分支设置`，不是工作区级配置中心。
3. 工作区设置、资源管理、偏好、模型配置在当前代码里仍是并列入口，尚未完全收成一处。
4. 如果后续还想重新设计统一入口，应先明确是要合并入口，还是只统一文案与导航，不要把未实现的总入口提前写成既成事实。
