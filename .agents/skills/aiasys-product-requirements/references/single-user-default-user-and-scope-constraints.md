# 单机个人模式默认用户与作用域约束

状态: 主线专题
更新时间: 2026-04-04
适用版本: v0.3.9 当前主线

---

## 1. 文档目标

本文档用于把 AIASys 当前单机个人模式下的“默认用户”和“作用域边界”写清楚。

它回答的是：

- 当前为什么允许保留默认本地用户
- 用户、任务、runtime 在这一阶段分别承担什么作用
- 哪些多人协作语义当前不应继续推进成主线

---

## 2. 核心结论

### 2.1 当前主线是单机个人模式

当前产品默认交付形态不是团队协作平台，而是单机个人工作台。

### 2.2 默认本地用户只是兼容骨架

当前后端允许保留一个默认本地用户，例如：

- `local_default`

它的职责是：

- 兼容现有 `user_id / session_id` 路径结构
- 兼容知识库、工作区和数据目录隔离逻辑
- 让本地单机模式不必先引入完整登录系统

### 2.3 当前作用域优先收敛为三层

当前主线优先只强调以下三层：

- 系统目录层
- 任务工作区层
- runtime 证据层

用户层在这一阶段仍然存在，但更像内部兼容骨架和默认值来源，而不是用户必须理解的继承体系核心。

---

## 3. 当前作用域约束

### 3.1 system

回答：

- 系统提供什么
- 有哪些市场条目和模板

### 3.2 user

回答：

- 默认用户的长期默认配置
- 数据隔离与目录归属

### 3.3 task

回答：

- 当前任务工作区怎么跑
- 当前任务挂载什么资源与能力

### 3.4 runtime

回答：

- 当前这一轮实际用了什么
- 当前 runtime 是否 ready / busy / refresh_required

---

## 4. 当前不继续推进为主线的方向

当前阶段不应继续把以下语义推进成前台主骨架：

- 团队共享工作区
- 团队级 MCP 继承链
- 多租户登录体系
- 项目空间与团队空间并列主导航

---

## 5. 对实现的直接约束

### 5.1 后端约束

- 可以保留默认本地用户兼容逻辑
- 不要为了当前阶段去强行移除所有 `user_id`
- 但也不要继续把多用户继承模型扩得更复杂

### 5.2 前端约束

- 不再把“登录 / 团队 / 租户”作为当前主语义
- 设置中心优先解释为个人长期默认配置中心
- 任务工作区优先解释为当前工作的主对象

### 5.3 文档约束

- 任何新文档若提到 `user`，都应先说明它当前是兼容骨架还是产品主语义
- 不要把未来多人形态写成当前已经承诺的主线

---

## 6. 当前建议结论

当前主线应统一以下口径：

1. AIASys 当前默认面向单机个人使用。
2. 默认本地用户只是实现兼容骨架。
3. 文档和前端优先突出任务工作区，而不是用户体系本身。
4. 后续如进入多人模式，应作为新阶段独立设计，不在当前主线中偷渡扩张。
