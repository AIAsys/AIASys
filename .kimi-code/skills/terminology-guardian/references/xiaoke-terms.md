## 身份定位

### AI时代的探索者

| 字段 | 内容 |
|------|------|
| 标准描述 | AI时代的探索者 |
| 使用场景 | 自我介绍、文章署名 |
| 定义 | 专注于AI应用落地，关注大模型、Agent、RAG系统开发的实践者 |
| 来源 | 个人品牌定位 |
| 示例句 | 我是筱可，一个AI时代的探索者。 |
| 备注 | 主身份标签，避免使用学生气表达 |

### DataWhale成员

| 字段 | 内容 |

### 独立开发者

| 字段 | 内容 |
|------|------|

| 使用场景 | 项目复盘、技术分享 |

## 原创概念

### 上下文工程 / Context Engineering

| 字段 | 内容 |
|------|------|
| 标准描述 | 上下文工程 |
| 英文 | Context Engineering |
| 使用场景 | AI协作、编程方法 |
| 定义 | 系统性地设计、组织和管理AI协作中上下文信息的方法论 |
| 来源 | 原创概念，区别于「提示词工程」 |
| 示例句 | AI编程时代，上下文工程比提示词工程更重要。 |
| 备注 | 核心差异化概念，需要统一使用 |

### 渐进式开发 / Iterative Development

| 字段 | 内容 |
|------|------|
| 标准描述 | 渐进式开发 |
| 使用场景 | 开发方法、项目复盘 |
| 定义 | 通过多次小规模迭代逐步完善产品的开发方式，强调可控和可验证 |
| 来源 | 个人开发方法论，文章《我就爱用AI写笨代码》 |
| 示例句 | 我推崇渐进式开发，每次改动一个可验证的小功能。 |
| 备注 | 与「敏捷开发」区分，更强调个人掌控感 |

### 笨代码 / Dumb Code

| 字段 | 内容 |
|------|------|
| 标准描述 | 笨代码 |
| 使用场景 | 编程哲学、代码实践 |
| 定义 | 显式、可读、易于理解的代码，与过度抽象的「聪明代码」相对 |
| 来源 | 文章《我就爱用AI写笨代码》 |
| 示例句 | AI让写笨代码变得更容易，因为你可以快速迭代。 |
| 备注 | 有特定含义的概念，非贬义 |
