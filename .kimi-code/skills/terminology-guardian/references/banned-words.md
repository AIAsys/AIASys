## 商业黑话（全场景禁用）

以下词汇在任何场景都不使用：

| 禁用词 | 替代方案 | 禁用原因 |
|--------|----------|----------|
| 赋能 | 帮助、支持、提升 | 空洞无意义 |
| 抓手 | 切入点、关键 | 生硬翻译 |
| 闭环 | 完整流程、循环 | 被过度使用 |
| 对齐 | 统一、确认、同步 | 太 corporate |
| 落地 | 实现、完成 | 空话 |
| 痛点 | 问题、需求 | 套话 |
| 价值主张 | 核心价值、定位 | 过于学术 |
| 商业模式画布 | - | 不需要全称 |
| 生态 | 系统、体系 | 被滥用 |
| 矩阵 | 组合、体系 | 故作高深 |
| 颗粒度 | 细节程度、精度 | 不必要术语 |
| 底层逻辑 | 本质、核心原理 | 空话 |
| 顶层设计 | 整体规划 | 过于宏大 |
| 组合拳 | 多种方法配合 | 口语化过度 |
| 引爆点 | 关键点、转折点 | 夸张 |

## AI味表达（写作中禁用）

以下表达是AI生成内容的典型痕迹：

| 禁用表达 | 替代方案 | 说明 |
|----------|----------|------|
| 值得注意的是 | 直接陈述事实 | AI常用过渡 |
| 首先...其次...最后... | 重新组织句子结构 | 过于机械 |
| 综上所述 / 总之 | 删除，或用「核心结论」 | 套话 |
| 从某种程度上说 | 删除，直接说观点 | 模糊化 |
| 不可否认的是 | 删除，直接陈述 | 无效强调 |
| 这是一个复杂的问题 | 直接展开分析 | 拖延 |
| 让我们来看看 | 直接进入内容 | 多余的引导 |
| 在本文中，我们将探讨 | 删除，直接说主题 | 学术腔 |
| 随着...的发展 | 具体说明变化 | 陈词滥调 |
| 它不仅...而且... | 拆分句子 | 句式套路 |

## 过度修辞（观点文中慎用）

以下修辞手法容易显得刻意：

| 类型 | 禁用示例 | 建议 |
|------|----------|------|
| 排比堆砌 | 「它是...它是...它是...」 | 最多一组，用于关键处 |
| 过度引用 | 每段一个名人名言 | 用自己观点代替 |
| 强制押韵 | 为了押韵牺牲准确 | 放弃押韵 |
| 数字标题党 | 「3个秘密」「5个真相」 | 除非真有清单 |
| 情绪夸张 | 「所有人都错了」「颠覆认知」 | 用数据说话 |

## 模糊形容词（尽量具体化）

以下形容词尽量不用，改用具体描述：

| 禁用 | 替代方案 |
|------|----------|
| 非常 | 具体程度或数据 |
| 很 | 删除或具体化 |
| 特别 | 说明特殊之处 |
| 比较 | 给出对比标准 |
| 大概 / 大约 | 给出精确范围 |
| 一定程度上 | 说明程度 |
| 明显的 | 描述具体现象 |
| 重要的 | 说明重要性体现在哪 |

## 场景特定禁用

### 技术教程中禁用

- ❌ 「很简单」→ 用「步骤如下」
- ❌ 「只要...就...」→ 用「需要...然后...」
- ❌ 「一键」→ 用「一键配置」或具体说明
- ❌ 「秒懂」→ 用「快速理解」或删除

### 观点文章中禁用

- ❌ 「我认为」→ 用「我的观察是」「数据表明」
- ❌ 「可能」「也许」过度使用→ 确定的说确定的，不确定的说清楚不确定
- ❌ 「众所周知」→ 直接陈述事实
