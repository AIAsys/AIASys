## 开发模式

### 渐进式开发 / Iterative Development

| 字段 | 内容 |
|------|------|
| 标准用词 | 渐进式开发 |
| 英文 | Iterative Development |
| 同义词 | 迭代开发 |
| 禁用词 | 敏捷开发、小步快跑、逐步开发 |
| 使用场景 | 技术文、项目复盘 |
| 定义 | 通过多次小规模迭代逐步完善产品的开发方式 |
| 示例句 | 我更喜欢渐进式开发，每次只改动一个可验证的小功能。 |
| 备注 | 你文章中的特色概念，与「瀑布式」相对 |

### 上下文工程 / Context Engineering

| 字段 | 内容 |
|------|------|
| 标准用词 | 上下文工程 |
| 英文 | Context Engineering |
| 同义词 | - |
| 禁用词 | 提示工程、语境设计 |
| 使用场景 | 技术文、观点文 |
| 定义 | 系统性地设计、组织和管理AI协作中上下文信息的方法论 |
| 示例句 | AI编程时代，上下文工程比提示词工程更重要。 |
| 备注 | 你的原创概念，需要统一使用 |

### 脚手架 / Scaffolding / Boilerplate

| 字段 | 内容 |
|------|------|
| 标准用词 | 脚手架 / Scaffolding |
| 英文 | Scaffolding / Boilerplate |
| 同义词 | 模板代码、项目模板 |
| 禁用词 | 框架、样板 |
| 使用场景 | 技术教程 |
| 定义 | 项目启动时的基础代码结构和配置 |
| 示例句 | 好的脚手架可以让你5分钟启动新项目。 |
| 备注 | Boilerplate指重复代码，Scaffolding指生成结构 |

### 原语 / Primitive

| 字段 | 内容 |
|------|------|
| 标准用词 | 原语 / Primitive |
| 英文 | Primitive |
| 同义词 | 基础构件 |
| 禁用词 | 原生、原始 |

### 幂等 / Idempotent

| 字段 | 内容 |
|------|------|
| 标准用词 | 幂等 / Idempotent |
| 英文 | Idempotent |
| 同义词 | - |
| 禁用词 | 幂等性、可重复执行 |
| 使用场景 | 技术文 |
| 定义 | 同一操作执行多次与执行一次效果相同 |
| 示例句 | 接口设计应该保证幂等，避免重复提交导致数据错误。 |
| 备注 | 可作形容词：幂等的 |

### 声明式 / Declarative

| 字段 | 内容 |
|------|------|
| 标准用词 | 声明式 |
| 英文 | Declarative |
| 同义词 | - |
| 禁用词 | 声明性、描述式 |

## 工具与方法

| 英文 | Minimum Viable Product |
| 同义词 | 最小可用产品 |
