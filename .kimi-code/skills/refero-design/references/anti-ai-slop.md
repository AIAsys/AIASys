# Anti-AI-Slop Manifesto

**CRITICAL: Your design must NOT look AI-generated.**

AI-generated interfaces converge on the same tired patterns because they optimize for "safe" and "average." Real designers make intentional, contextual choices.

---

## Critical: THE SINGLE BIGGEST AI TELL: INDIGO/VIOLET

**If you remember nothing else from this guide, remember this:**

Every AI model defaults to indigo/violet (`#6366f1`, `#8b5cf6`, `#7c3aed`). It's become the universal fingerprint of "this was generated by AI."

**Why it happens:**
- Training data includes millions of examples using Tailwind's indigo
- Purple is perceived as "modern" and "safe"
- LLMs optimize for the average, and indigo IS the average

**The rule:** 
> **NEVER use indigo/violet unless the brand explicitly requires it.**

If someone asks "why purple?" and your answer is "it looked nice" or "I don't know"—you've failed. Change the color.

**Quick alternatives:**
| Instead of | Try | Feeling |
|------------|-----|---------|
| Indigo `#6366f1` | Blue `#2563eb` | Trust, professional |
| Violet `#8b5cf6` | Teal `#0d9488` | Fresh, distinctive |
| Purple `#7c3aed` | Brand color | Authentic, intentional |

## What Makes Design Look Generic

**Typography symptoms:**
- Bad: Using the same font as every other AI-generated site
- Bad: Same weight throughout (no hierarchy)
- Bad: Default system fonts without intention
- Bad: No distinction between display and body text
- Bad: **Missing letter-spacing** (especially on ALL CAPS and small text)

**Color symptoms:**
- Bad: **Default indigo/violet** (#1 indicator)
- Bad: Gradients that don't serve function
- Bad: Perfectly even color distribution
- Bad: Colors chosen for "safety" not meaning
- Bad: No clear accent hierarchy

**Layout symptoms:**
- Bad: Perfectly symmetrical everything
- Bad: Cookie-cutter card grids
- Bad: Hero with left text + right image (every landing page ever)
- Bad: Centered everything with no visual tension

**Visual symptoms:**
- Bad: Abstract blob backgrounds (no meaning)
- Bad: Generic 3D illustrations
- Bad: Effects without purpose
- Bad: Stock imagery that could be anywhere

---

## The Antidote: Intentional Design

**Typography with purpose:**
- Good: Choose fonts that **match your tone** from references
- Good: Create **clear hierarchy** (3-4 distinct levels)
- Good: Use **weight and spacing** to differentiate, not just size
- Good: Study what fonts **successful apps in your space** use

**Color with meaning:**
- Good: Build palette **from your references**
- Good: Use **dominant + sharp accent** (not evenly distributed)
- Good: Make semantic colors **actually semantic**
- Good: Consider **dark mode from the start**

**Layout with intention:**
- Good: Create **visual tension** through asymmetry
- Good: Use **whitespace as a design element**
- Good: Break the grid **intentionally** (one element, not everything)
- Good: Guide the eye with **clear visual hierarchy**

**Details that distinguish:**
- Good: Custom illustrations or **real photography**
- Good: Micro-interactions that **reinforce brand**
- Good: Shadows and depth **when they serve hierarchy**
- Good: One memorable detail users will **actually remember**

## The AI Slop Detector Checklist

```
□ Is the accent color NOT indigo/violet? (instant fail if unjustified)
□ Does ALL CAPS text have letter-spacing?
□ Would this pass the "screenshot test" next to references?
□ Are font choices intentional and contextual?
□ Are colors derived from research, not defaults?
□ Does the layout have visual interest and tension?
□ Are there any generic patterns you can't justify?
□ Could you explain WHY each design choice was made?
```

## Experiment vs. Safe

**"Best practices" can be a trap.**

AI models default to the most common patterns because they're trained on averages. But the most memorable designs often break conventions intentionally.

**Safe (forgettable):**
- 3-column pricing grid because "everyone does it"
- Hero with left text + right image because "it works"
- Card grid with equal spacing because "it's clean"

**Intentional (memorable):**
- 2-column pricing because your product only has 2 tiers
- Full-width hero image with overlay because it fits the brand
- Asymmetrical layout because you want visual tension

**The difference:** Can you justify the choice? If yes—do it. If no—question it.

> "I chose this because [specific reason for THIS project]" beats "I chose this because everyone does it."
